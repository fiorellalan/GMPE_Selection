compute_targetspectra.py
compute_all_targetspectra.py

Compute GMPE target response spectra for spectral matching —
single-GMPE or multi-GMPE ensemble with mean / median / fractiles.

⚡ Usage

# Single GMPE (e.g. AbrahamsonEtAl2014)
python compute_targetspectra.py configs/config_HF_SMS_Ab2014.json

# All GMPEs together (mean, median, q16, q84)
python compute_all_targetspectra.py configs/config_all_GMPE.json

# Generate the 'all GMPEs' config first
python generate_configs.py --all

📐 The Two Scripts

compute_targetspectra.py
Single GMPE — reads a config with one GMPE (e.g. config_HF_SMS_Ab2014.json), computes the target spectrum via OpenQuake, saves the .tgt file and a PDF figure.

Used with: per-GMPE configs from generate_configs.py.
compute_all_targetspectra.py
All GMPEs — reads a config listing all 15 GMPEs (config_all_GMPE.json), computes each one, then calculates the mean, median, q16, q84 across the ensemble.

Used with: config_all_GMPE.json (generated via generate_configs.py --all).

🧩 What It Does

  1. OpenQuake environment — auto-switches to ~/openquake/bin/python if needed (no manual source activate required).
  2. Reads config — loads the SCENARIO block: event magnitude, depth, epicentral distance, Vs30, rake, dip, and the list of GMPE names.
  3. Computes reference spectrum — Berge-Thierry (2003) for the reference + site-specific non-ergodic adjustment factor.
  4. Computes GMPE spectra — for each GMPE in the list, calls OpenQuake's GSIM library with the scenario parameters. Returns spectral acceleration (SA) at frequencies from 0.5 to 33 Hz.
  5. Generates .tgt file — writes the target spectrum in RSPMatch format (3 header lines, then frequency / dummy / dummy / SA per line). SA is in m/s².
  6. Generates PDF figure — log-log plot of all GMPE curves + BT2003 reference.

For compute_all_targetspectra.py, step 4 additionally computes the ensemble mean, median, 16th and 84th percentiles across all GMPEs, and saves a separate target{event}_MEAN.tgt.

🔬 Algorithm Trace

Step-by-step logic of compute_targetspectra.py and compute_all_targetspectra.py at runtime:

Phase 1 — OpenQuake environment

  1. Check Python: compare sys.executable against ~/openquake/bin/python.
  2. Auto-switch: if not already running from the OpenQuake environment, call os.execv() to restart the script using ~/openquake/bin/python with the same arguments. This ensures all OpenQuake GSIM libraries are available.
  3. No manual activation needed: simply run with any Python — the script self-restarts in the correct environment.

Phase 2 — Load scenario

  1. Read config JSON: load the SCENARIO block containing event parameters (magnitude, depth, epicentral distance, Vs30, rake, dip) and the list of GMPE names.
  2. Compute distances: RJB = Epi, Rrup = √(ztor² + RJB²), width from magnitude/rake-dependent scaling relations.
  3. Build site/rupture/distances contexts: populate OpenQuake SitesContext, RuptureContext, DistancesContext with the scenario parameters (Vs30, magnitude, rake, dip, depths, distances).

Phase 3 — Compute reference spectrum

  1. Berge-Thierry (2003): compute the BT2003 reference GMPE at 100 log-spaced frequencies from 0.25 to 33 Hz using the magBT / depthBT scenario parameters.
  2. Site-specific non-ergodic factor: SND = 1.5 × BT2003 — a simplified adjustment.
  3. Site amplification factors (SMF): hardcoded frequency-dependent factors for French hard-rock sites.

Phase 4 — Compute GMPE spectra

  1. For each GMPE: loop over all GMPE names in the config's GMPElist.
  2. Call OpenQuake GSIM: instantiate the GSIM class from openquake.hazardlib.gsim.get_available_gsims(), then call get_mean_and_stddevs() with the site/rupture/distances contexts at each frequency.
  3. Frequency grid: 100 log-spaced frequencies from 0.5 to 33 Hz. Some GMPEs (Kotha 2020) use a custom period grid with interpolation.
  4. Collect SA: store the median spectral acceleration (in m/s²) at each frequency for each GMPE.

For compute_all_targetspectra.py, additionally computes the mean, median, 16th and 84th percentiles across the GMPE ensemble at each frequency.

Phase 5 — Save target files

  1. Create directory: Input_{event}_{code}/ per GMPE (e.g. Input_HF_SMS_Ab2014/).
  2. Write .tgt file: 3 header lines (filename, npts+flag, damping ratio) followed by one line per frequency: freq 0 1000 SA where SA is in m/s².
  3. For the all-GMPE script: also saves a Input_{event}_MEAN/target_{event}_MEAN.tgt with the ensemble mean spectrum.

Phase 6 — Generate PDF figure

  1. Log-log plot: create a figure with one subplot per event.
  2. Plot curves: BT2003 reference in black, each GMPE curve coloured by event with legend.
  3. Save: Figures/Alea_{events}events.pdf.

🔐 OpenQuake Environment

Both scripts automatically switch to the OpenQuake Python environment if needed:

# At the top of each script:
if sys.executable != "~/openquake/bin/python":
    try:
        import openquake
    except ImportError:
        os.execv("~/openquake/bin/python", [...] + sys.argv)

No manual source activate needed. Simply run with your system Python and it self-restarts in the correct environment.

⚠ The environment at ~/openquake/ must have openquake.engine, scipy, numpy, matplotlib installed.

# 1. Generate configs (one per GMPE + one 'all')
